1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
	git checkout . 

--------------------------------------------------------------------------------------------------
2.Как вернуть файл из индекса в рабочий каталог?
	git restore --staged file.txt 

--------------------------------------------------------------------------------------------------
3.Как отменить ошибочный commit ?
	git commit --amend -m "new commit"

---------------------------------------------------------------------------------------------------
4.Как исправить коммит отправленный на сервер GitHub?
	git reset HEAD^    || (~3)
	
---------------------------------------------------------------------------------------------------
5.Как создать новую ветку и переключится в нее?
	git checkout -b newBranch 

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
5.В чем отличение между fast-forward merge  and non fast-forfard merge?
	Если одна из веток не изменялась, а работы проводились только на одной из веток, то происходит fast-forward merge, это быстое слияне веток, гит просто передвигает последний коммит (указатель HEAD) с ветки которая не изменялась на последний коммит ветки куда вливаемся.
	----
	Если производились изменения файлов на всех ветках, то при слиянии происходит non fast-forfard merge, гит сам создает новый коммит и в нем объединяет все изменения из обоих веток.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
	Если производилась работа над одним файлом с разных веток, тогда возникает конфликт. Гит подскажет где возник конфликт, нужно зайти в этот файл и оставить нужные изменения. затем нужно сделать коммит.
